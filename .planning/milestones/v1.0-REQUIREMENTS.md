# Requirements: Engines of Experience (v1.0)

**Defined:** 2026-01-29
**Core Value:** Consistent output of creative atoms that compound into a body of work, tools, and audience — producing results, not consuming time learning tools.

## v1 Requirements — ALL SHIPPED ✓

### Monorepo Structure

- [x] **REPO-01**: Single monorepo contains all atoms, compositions, tools, and notes — ✓ v1.0
- [x] **REPO-02**: Standardized atom directory structure (each atom is self-contained folder) — ✓ v1.0
- [x] **REPO-03**: Shared tooling config (TypeScript, Vite, linting) at root level — ✓ v1.0
- [x] **REPO-04**: Git + LFS configured for media assets (audio, video, images) — ✓ v1.0

### Creative Coding (Visual)

- [x] **VIS-01**: Create p5.js sketches with Vite hot-reload in browser — ✓ v1.0
- [x] **VIS-02**: Atom template scaffolding (one command creates new sketch with boilerplate) — ✓ v1.0
- [x] **VIS-03**: Standardized atom interface (init, update, draw, cleanup, config) — ✓ v1.0
- [x] **VIS-04**: Parameter config per atom (colors, speeds, sizes) as editable JSON — ✓ v1.0

### Audio

- [x] **AUD-01**: Create audio atoms with Tone.js in browser — ✓ v1.0
- [x] **AUD-02**: Simple tune composition (synths, sequences, effects) — ✓ v1.0
- [x] **AUD-03**: Audio-reactive visual parameters (FFT analysis drives sketch properties) — ✓ v1.0
- [x] **AUD-04**: Audio atom template scaffolding — ✓ v1.0

### Video Capture

- [x] **VID-01**: Capture running atom canvas to video via Playwright — ✓ v1.0
- [x] **VID-02**: FFmpeg encoding to platform formats (16:9 YouTube, 9:16 TikTok) — ✓ v1.0
- [x] **VID-03**: Thumbnail extraction from key frames — ✓ v1.0
- [x] **VID-04**: Capture with audio (record Tone.js output alongside visuals) — ✓ v1.0

### CLI Cockpit

- [x] **CLI-01**: `eoe create <type> <name>` scaffolds new atom from template — ✓ v1.0
- [x] **CLI-02**: `eoe dev <atom>` starts Vite dev server with hot-reload — ✓ v1.0
- [x] **CLI-03**: `eoe build <atom>` produces production bundle — ✓ v1.0
- [x] **CLI-04**: `eoe capture <atom>` records canvas to video — ✓ v1.0
- [x] **CLI-05**: `eoe list` shows all atoms with status (idea/wip/done) — ✓ v1.0

### Creative Notebook

- [x] **NOTE-01**: Markdown-based idea capture (quick jot with context, tags) — ✓ v1.0
- [x] **NOTE-02**: Per-atom notes file tracking stage, plans, creative decisions — ✓ v1.0
- [x] **NOTE-03**: WIP tracker showing all pieces and their current stage — ✓ v1.0
- [x] **NOTE-04**: CLI integration (`eoe note <atom>` opens notes, `eoe status` shows WIP) — ✓ v1.0

## Milestone Summary

**Shipped:** 25 of 25 v1 requirements ✓

**Adjusted during implementation:**
- TikTok publishing removed (complexity deferred to v2) — scope reduced to YouTube-only for v1
- Manual publishing workflow maintained (no automation) per constraint
- Composition layer simplified to manifest-based orchestration (not full DAW)

**Dropped:** None — all v1 requirements either shipped or deferred to v2

**Tech additions:**
- Playwright ^1.50.0 (video capture automation)
- fluent-ffmpeg ^2.1.3 (video encoding wrapper)
- googleapis ^144.0.0 (YouTube API)
- exponential-backoff ^3.1.1 (retry logic)
- open ^10.0.0 (OAuth browser flow)

**Code delivered:**
- lib/audio/ — Tone.js synths, effects, disposal patterns
- lib/capture/ — Playwright browser automation, MediaRecorder injection
- lib/encoding/ — FFmpeg wrapper with aspect ratio handling, thumbnail extraction
- lib/platforms/ — YouTube OAuth2, resume upload, credential storage
- cli/commands/ — create, dev, build, capture, list, note, status, completion, auth, publish
- atoms/ — Visual and audio templates with HMR cleanup patterns
